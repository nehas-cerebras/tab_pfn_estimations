The two phases, block by block

There are 3 layers. Each layer = block1 (rows → 128×128 summary) then block2 (summary → rows). Goal: see exactly why fit runs block1 ×3 and block2 ×5.

block1 — reads all rows, makes a summary block2 — gives each row an embedding cached summary (kept) train rows only ALL rows (train+test)

🧮 The tally — where ×3 and ×5 come from

block1  =  3

L0L1L2
3 (Phase 1) + 0 (Phase 2, all cached) = 3

Runs only in Phase 1, once per layer, always over the N train rows.

block2  =  5

L0L1 | L0L1L2
2 (Phase 1, train rows) + 3 (Phase 2, all rows) = 5

Solid = Phase 1 (only to chain into the next block1). Faded = Phase 2 (the real per-row embeddings, over all rows).

Why block2 appears twice for the train rows (once each phase): Phase 1 needs block2 just to hand the next layer's block1 a correctly-transformed input; Phase 2 re-runs block2 to actually produce every row's output embedding. Trading a little recompute for not having to store all the intermediate train activations.
